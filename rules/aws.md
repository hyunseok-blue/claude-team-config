# AWS CLI Agent 유의사항

## 필수 확인 사항
- `--region` 플래그를 항상 명시적으로 지정 (기본 리전 의존 금지)
- 프로덕션 계정과 개발 계정을 반드시 구분하여 작업

## 절대 금지 사항
- **IAM 권한(정책, 역할) 추가/삭제/수정 절대 금지** — 권한 변경이 필요하면 사용자에게 안내만 할 것
- 사용자 확인 없이 리소스 삭제/종료 명령 실행 금지 (delete, terminate, remove)
- IAM에 와일드카드(`*`) 권한 부여 금지 — 최소 권한 원칙 준수
- S3 버킷 퍼블릭 접근 설정 금지 (명시적 요청 없는 한)
- 보안 그룹에 0.0.0.0/0 인바운드 규칙 추가 금지 (SSH, RDP 등)
- 시크릿, 액세스 키 등 민감 정보를 로그나 출력에 노출 금지

## 비용 관련 주의
- NAT Gateway, ELB, RDS Multi-AZ 등 고비용 리소스 생성 시 반드시 비용 경고
- EC2 인스턴스 타입 선택 시 비용 영향 안내
- 불필요한 리소스는 작업 완료 후 정리 방안 제시

## 작업 패턴
- 파괴적 작업 전 `--dry-run` 플래그 사용 권장
- 복잡한 인프라 변경은 CloudFormation/IaC 방식 권장
- 명령 실행 후 결과 검증 명령 함께 제시
- 에러 발생 시 `--debug` 플래그로 상세 진단
