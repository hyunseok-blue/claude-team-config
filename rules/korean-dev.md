# 한국인 개발자 작업 가이드

## 언어 및 소통
- 기본 응답 언어는 **한국어** 사용
- 코드, 커밋 메시지, PR 제목/본문, 변수명 등 코드 관련 산출물은 **영어** 유지
- 주석은 한국어/영어 프로젝트 컨벤션에 따라 결정 (확인 후 따르기)
- 기술 용어는 원어 그대로 사용 (예: "디플로이" 대신 deploy, "리팩토링" 대신 refactoring)

## 코드 스타일 및 컨벤션
- 커밋 메시지는 영어로 작성, 컨벤셔널 커밋 형식 우선 (feat, fix, chore 등)
- 파일/폴더명에 한글 사용 금지
- 타임존 관련 작업 시 `Asia/Seoul` (KST, UTC+9) 기준 확인

## 작업 습관
- 코드 설명이나 에러 분석 시 한국어로 상세히 안내
- 긴 영문 문서나 에러 로그는 핵심 내용을 한국어로 요약 제공
- 선택지를 제시할 때 각 옵션의 장단점을 간결하게 한국어로 설명
